# 1. X 윈도
 
**MIT** : X 윈도 서버로 사용되는 X.org에 적용된 라이선스    
**X11**: 현재 배포되고 있는 x.org의 버전

### X 윈도의 구조
클라이언트/서버 구조    
서로간의 통신을 위해 **`X Protocol`** 사용    
기본구조가 네트워크 기반이기 때문에 로컬의 사용자가 키보드나 마우스로 입력하면 **TCP 포트 `6000`번**을 통해 전달되고, 관련 X 클라이언트 프로그램이 실행됨

### X 윈도의 실행
1) 부팅시에 X 윈도를 실행하는 방법 (Runlevel 5) - **그래픽 모드** → **`디스플레이 매니저`**
2) **텍스트 모드**에서 실행하는 방법 (Runlevel 3) - `startx` 명령어   
- `/etc/inittab`: X 윈도 실행하기 위해 리눅스의 부팅 모드 설정 파일
- `id:5:initdefault:`: X 윈도가 실행되도록 관련 설정 파일을 수정하려고 할 때 들어갈 내용 

※ `systemctl set-default graphical.target`: 시스템 시작시 **x 윈도 모드로 부팅이 되도록** 설정하는 명령

### 디스플레이 매니저 (Display Manager)

ex)
- **`XDM`** (X Display Manager)
- **`GDM`** (GNOME Display Manager)
- **`KDM`** (KDE Display Manager)

### `DISPLAY`

`# export DISPLAY="192.168.10.55:1.2"` : X 클라이언트 프로그램을 192.168.10.55의 두번째 실행된 X서버의 세번째 모니터로 전송

### TCP/IP 
1960년대 말 미국방성에서 연구를 시작, 1980년대 초에 공개되어 **컴퓨터 기종에 상관없이 정보 교환이 가능하게** 해주는 통신 프로토콜
- 구조
  - ④ 응용 계층
  - ③ 전송 계층: TCP, UDP
  - ② 인터넷 계층: IP, ICMP, ARP
  - ① 네트워크 인터페이스 계층   

### TCP 
연결 지향 프로토콜    
세그먼트가 응답(Ack)을 주고받으며 점검 → 오류 검사 및 제어     
- +) 최다 경로를 찾아 1:1 직접연결해 패킷을 주고받음 → 전송/오류수정 신뢰도 높음
- +) 안정성, 신뢰성 우수
- -) 많은 전송시간    
ex) 사용자 데이터 전송, 대용량 전송

#### 3-way handshaking 수행 순서
`SYN` → `ACK/SYN` → `ACK`

### UDP
비연결 전송 프로토콜   
세그먼트를 보내기만하고 응답(ACK)을 주고받지 않음   
→ 제대로 전달됐는지 확인하지 않고, 오류수정도 안함
- +) 속도 빠름, 직접 연결에 따른 네트워크 부하를 줄임    
ex) 시스템 내부 메세지 전달, 소규모 데이터 전송, DNS

### OSI 7계층
- ⑦ 응용 계층 (data) 
- ⑥ 표현 계층 (data)
- ⑤ 세션 계층 (data)
  - `SSL` 
- ④ 전송 계층 (segment)
- ③ 네트워크 계층 (packet)
  - `IP`, `ICMP`, `ARP` ... 
- ② 데이터 링크 계층 (frame)
- ① 물리 계층 (bit)

### 특수 목적 IP 주소
#### 사설 IP 주소
- A 클래스: `10.0.0.0` ~ `10.255.255.255` → 네트워크 개수 1개

### 인터넷 서비스 종류
- **FTP (File Transfer Protocol)**
  - **TCP/IP 기반** 서버와 클라이언트 사이에서 파일을 주고받기 위한 프로토콜 
  - FTP를 사용하기 위해 FTP 서버가 반드시 필요
  - 익명 계정(Anonymous)을 이용해 접속 가능
  - Active 모드와 Passive 모드를 지원
  - `put`, `recv`: 지정한 파일 **로컬 시스템으로 가져오는** FTP 명령어
  - `put`, `send`: 지정한 파일을 **원격시스템으로 보내는** FTP 명령어
  - `mput`: 로컬시스템에 있는 **여러 개의 파일**을 동시에 보낼 때 사용
  - `mget`: 로컬시스템에 있는 **여러 개의 파일**을 가져올 때 사용
- **SSH**
  - 인증파일 경로: `/home/(사용자명)/.ssh/authorized_keys`
  - 옵션 `-l`: 다른 계정으로 접속할 때 사용
  - 옵션 `-p` : ssh서버의 포트번호가 22가 아닌 다른 포트를 사용할 경우
  - 관련 명령어: `ssh`, `scp`, `sftp`, `slogin`, `rsh` 
- **삼바 (Samba)** 
  - 리눅스 → 윈도우 
  - SMB 프로토콜 확장 => **`CIFS`** (Common Internet File System)   
- **NFS (Network File System)**
  - 리눅스 → 리눅스 
  - 네트워크 상에서 다른 컴퓨터의 파일 시스템을 마운트하고 공유
  - `NIS`와 `RPC` 프로토콜 기반 작동 → 해당 서비스를 해주는 데몬을 먼저 실행해야 함 = `rpcbind 데몬`(구 `portmap 데몬`)

### 서브넷 마스크 (Subnet Mask)
= 네트워크 주소

#### 넷마스크 값(서브넷 마스크)과 호스트 수 (C클래스 기준)
(1) 넷마스크값 = `255.255.255.0`    
- **실제 네트워크에 사용** 가능한 IP 주소 = **254개**    
- 256 - 2(네트워크주소, 브로드캐스트주소) = 254    
  
(2) 넷마스크값 = `255.255.255.128`
- **실제 네트워크에 사용** 가능한 IP 주소 = **254개**    
- 서브넷 = 2개 (128개씩)
- 256 - 2(네트워크주소, 브로드캐스트주소)*2 = 254    

(3) 넷마스크값 = `255.255.255.192` 
- **실제 네트워크에 사용** 가능한 IP 주소 = **248개**    
- 서브넷 = 4개 (64개씩)
- 256 - 2(네트워크주소, 브로드캐스트주소)*4 = 248    

→ 넷마스크값(서브넷마스크)는 서브 네트워크로 분할하면 각 네트워크에 네트워크 주소와 브로드캐스트 주소가 별도 요구되며, **인터넷에 연결할 경우** 각 네트워크별 `게이트웨이 주소`가 추가로 요구되므로 **실제 사용가능한 IP주소가 점점 줄어듦!!**

### 관련 명령어
- `ifconfig`: 로컬 시스템에 장착된 **이더넷 카드의 MAC 주소** 확인할 때 사용
- `arp`: 로컬 네트워크에 있는 **다른 시스템에 장착된** 이더넷 카드의 **MAC 주소**를 확인할 때 사용
- `netstat`: 네트워크 연결 상태 출력, 라우팅 테이블 정보, 서버에 접속한 클라이언트의 IP 주소 및 포트번호 확인    
- `ss`: 소켓 상태 출력, `netstat`와 유사
- `ip`: 설정 정보 출력 및 변경
  - `ip addr show`: IP 주소 정보 출력 
  

※ **DHCP 서버**: 자동으로 ip 주소 할당
